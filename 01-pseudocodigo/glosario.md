# Glosario de Pseudocódigo

Este glosario reúne términos básicos para estudiar pseudocódigo antes de pasar a programación básica.

## Conceptos principales

| Término | Significado |
|---|---|
| Algoritmo | Conjunto ordenado de pasos para resolver un problema. |
| Pseudocódigo | Forma de escribir algoritmos con lenguaje cercano al humano, sin depender de un lenguaje de programación real. |
| PSeInt | Herramienta usada para escribir, probar y aprender pseudocódigo. |
| Entrada | Dato que recibe el algoritmo. |
| Proceso | Operaciones o pasos que transforman los datos de entrada. |
| Salida | Resultado mostrado al finalizar el algoritmo. |
| Variable | Espacio con nombre que guarda un dato. |
| Constante | Valor que no cambia durante la ejecución del algoritmo. |
| Tipo de dato | Clasificación de un valor, por ejemplo entero, real, carácter o lógico. |
| Operador | Símbolo o palabra que permite realizar una operación. |
| Expresión | Combinación de valores, variables y operadores que produce un resultado. |

## Estructuras de control

| Término | Significado |
|---|---|
| Secuencia | Instrucciones ejecutadas una después de otra. |
| Condicional | Estructura que permite tomar una decisión según una condición. |
| `Si Entonces` | Estructura usada cuando una acción depende de una condición. |
| `Segun` | Estructura usada para elegir entre varias opciones posibles. |
| Bucle | Estructura que repite instrucciones. |
| `Para` | Bucle recomendado cuando se conoce la cantidad de repeticiones. |
| `Mientras` | Bucle que evalúa la condición antes de ejecutar. |
| `Repetir` | Bucle que ejecuta primero y evalúa la condición al final. |

## Variables de apoyo

| Término | Significado |
|---|---|
| Contador | Variable que aumenta o disminuye para contar repeticiones o elementos. |
| Acumulador | Variable que suma valores progresivamente. |
| Bandera | Variable lógica usada para indicar si ocurrió o no una condición. |
| Promedio | Resultado de dividir una suma entre la cantidad de datos. |
| Validación | Revisión que confirma si un dato cumple una regla. |

## Pruebas

| Término | Significado |
|---|---|
| Caso de prueba | Dato o escenario usado para comprobar un algoritmo. |
| Caso normal | Situación esperada en la que el algoritmo debería funcionar sin problemas. |
| Caso límite | Situación extrema o especial que puede revelar errores. |
| Error lógico | Error donde el algoritmo ejecuta, pero produce un resultado incorrecto. |
