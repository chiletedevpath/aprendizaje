# Publicación y producción

Este módulo conecta el aprendizaje técnico con la publicación responsable de un proyecto. La meta es saber preparar una aplicación antes de mostrarla en GitHub, en una web o como evidencia profesional.

## Qué se aprende aquí

- Preparar un proyecto para revisión pública.
- Separar configuración local de configuración publicable.
- Usar variables de entorno para datos sensibles.
- Crear instrucciones claras de ejecución.
- Revisar enlaces, capturas, datos de prueba y dependencias.
- Entender la diferencia entre demo, entrega académica y versión publicable.

## Ruta sugerida

1. Verifica que el proyecto se ejecute desde cero.
2. Limpia archivos generados que no deben versionarse.
3. Revisa que no existan contraseñas, tokens ni datos personales reales.
4. Documenta instalación, ejecución y propósito del proyecto.
5. Agrega capturas o portadas solo si ayudan a entender el resultado.
6. Publica con una versión mínima clara.
7. Registra mejoras futuras sin prometer funciones que todavía no existen.

## Checklist antes de publicar

- El README explica qué hace el proyecto.
- Los enlaces funcionan.
- Las rutas y comandos fueron probados.
- No hay `node_modules`, `target`, `dist`, `build`, `.env` ni archivos temporales.
- Los datos de ejemplo son ficticios.
- Las imágenes tienen uso autorizado.
- El estado del proyecto es honesto.

## Criterio de avance

Este módulo se considera logrado cuando puedes explicar qué se publica, por qué se publica y qué todavía no debe mostrarse como producto final.
