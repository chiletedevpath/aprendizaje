# Patrones creacionales

Ejercicios sobre patrones orientados a controlar la creacion de objetos.

## Contenido

- `builder`: construccion paso a paso de objetos.
- `abstract-factory`: creacion de familias de componentes relacionados.
- `factory`: creacion de familias de objetos sin acoplar el cliente a clases concretas.
- `prototype`: clonacion de objetos base.
- `singleton`: control de una unica instancia compartida.
- `spring-boot`: ejemplos del mismo enfoque aplicados en proyectos Spring Boot.

## Proposito

Comprender cuando conviene centralizar, encapsular o flexibilizar la creacion de objetos dentro de una aplicacion.

## Uso comunitario

Puede publicarse por patrones individuales si cada guia explica el problema, el patron usado y una comparacion con una implementacion sin patron.
