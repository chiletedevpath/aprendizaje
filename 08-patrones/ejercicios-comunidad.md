# Ejercicios para la comunidad

Estos retos complementan los ejercicios existentes de `08-patrones`.

## Principios SOLID

1. Separar una clase con demasiadas responsabilidades.
2. Agregar un nuevo tipo de reporte sin modificar la clase principal.
3. Revisar una jerarquia y detectar si una subclase rompe el comportamiento esperado.

## Patrones creacionales

1. Crear una fabrica para construir recursos de aprendizaje.
2. Usar Builder para crear una ficha tecnica de proyecto.
3. Explicar cuando Singleton es util y cuando puede ser riesgoso.

## Patrones estructurales

1. Adaptar un servicio externo de mensajeria a una interfaz interna.
2. Crear una fachada para simplificar varias operaciones de un sistema.
3. Comparar codigo antes y despues de aplicar Adapter o Facade.

## Casos integradores

1. Modelar un pedido con notificaciones y observadores.
2. Combinar Adapter y Observer sin depender de nombres de marcas externas.
3. Escribir una guia que explique problema, patron aplicado y resultado.

## Criterio para resolver

Antes de aplicar un patron:

1. Describe el problema real.
2. Identifica que cambia con frecuencia.
3. Evalua si un patron reduce o aumenta complejidad.
4. Implementa una version pequena.
5. Explica la decision tecnica con tus palabras.
