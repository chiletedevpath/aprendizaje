# Patrones estructurales

Ejercicios sobre patrones orientados a organizar relaciones entre clases, objetos y subsistemas.

## Contenido

- `adapter`: adaptacion entre interfaces incompatibles.
- `facade`: simplificacion de acceso a un conjunto de subsistemas.
- `spring-boot`: ejemplos aplicados en proyectos Spring Boot.

## Proposito

Comprender como reducir acoplamiento, ocultar complejidad interna y permitir que partes del sistema trabajen juntas sin depender directamente de detalles internos.

## Uso comunitario

Puede publicarse por patrones individuales si se redacta como caso propio, con problema, solucion y explicacion de la mejora de diseno.
