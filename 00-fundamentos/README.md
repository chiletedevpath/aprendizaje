# Fundamentos

Este módulo es la entrada a la ruta de aprendizaje de Chilete DevPath.

Antes de escribir mucho código, un desarrollador necesita aprender a estudiar, resolver problemas, usar herramientas básicas y documentar su avance. Por eso este módulo funciona como base para todo lo que viene después: pseudocódigo, programación, POO, estructuras de datos, bases de datos, frontend, backend y patrones.

## Objetivo

Construir una base ordenada para aprender desarrollo de software de forma progresiva, práctica y explicable.

## Temas principales

- Mentalidad de aprendizaje técnico.
- Uso básico de computadora, terminal y editor.
- Organización de carpetas y archivos.
- Git y GitHub como evidencia de progreso.
- Lectura de errores y depuración inicial.
- Método para resolver ejercicios.
- Documentación breve de prácticas y laboratorios.

## Estructura del módulo

- `guia-de-estudio.md`: orientación para estudiar y avanzar sin perder el orden.
- `ejercicios-propuestos/`: retos para practicar antes de mirar una solución.
- `ejercicios-resueltos/`: soluciones explicadas paso a paso.
- `laboratorios/`: prácticas integradoras pequeñas.

## Cómo usar este módulo

1. Lee la guía de estudio.
2. Resuelve los ejercicios propuestos sin copiar la solución.
3. Compara tu respuesta con los ejercicios resueltos.
4. Documenta qué aprendiste y qué error corregiste.
5. Avanza al módulo `01-pseudocodigo` cuando puedas explicar tu solución con tus propias palabras.

## Criterio de avance

Este módulo se considera logrado cuando puedes:

- Crear una carpeta de práctica con nombre claro.
- Explicar qué problema estás resolviendo.
- Separar entrada, proceso y salida.
- Usar Git para registrar avances.
- Escribir una nota breve sobre lo aprendido.
