# Fundamentos

Este módulo es la puerta de entrada a la ruta de aprendizaje de Chilete DevPath.

Antes de escribir código, una persona que aprende desarrollo de software necesita construir hábitos base: entender problemas, organizar archivos, estudiar con método, documentar avances, cuidar su bienestar y usar herramientas como Git, GitHub e IA con criterio.

## Objetivo

Construir una base ordenada, ética y sostenible para iniciar el aprendizaje técnico desde cero y avanzar hacia pseudocódigo, programación, POO, estructuras de datos, bases de datos, frontend, backend y patrones.

## Público objetivo

Este módulo está dirigido a:

- estudiantes que recién empiezan en programación
- personas autodidactas que quieren ordenar su aprendizaje
- miembros de la comunidad Chilete DevPath
- estudiantes que desean documentar su progreso de forma profesional

## Temas principales

- Mentalidad de aprendizaje técnico.
- Entrada, proceso y salida.
- Método para resolver ejercicios.
- Organización de carpetas y archivos.
- Uso básico de Git y GitHub como evidencia de progreso.
- Lectura inicial de errores.
- Documentación breve de prácticas y laboratorios.
- Uso responsable de IA.
- Autoría, fuentes y ética profesional.
- Bienestar, descanso y pausas durante el aprendizaje.

## Estructura del módulo

- `guia-de-estudio.md`: orientación para estudiar, practicar y documentar sin perder el orden.
- `glosario.md`: términos básicos para entender el módulo.
- `plantilla-evidencia.md`: formato base para registrar una práctica o laboratorio.
- `checklist-avance.md`: autoevaluación para saber si ya puedes avanzar a pseudocódigo.
- `ejercicios-propuestos/`: retos para practicar antes de mirar una solución.
- `ejercicios-resueltos/`: soluciones explicadas paso a paso.
- `laboratorios/`: prácticas integradoras pequeñas.

## Cómo usar este módulo

1. Lee la guía de estudio.
2. Consulta el glosario si encuentras términos nuevos.
3. Revisa las políticas de Chilete DevPath.
4. Resuelve los ejercicios propuestos sin mirar primero las soluciones.
5. Compara tu respuesta con los ejercicios resueltos.
6. Documenta tu práctica usando la plantilla de evidencia.
7. Haz pausas activas si tu sesión se vuelve larga.
8. Completa el checklist de avance.
9. Avanza al módulo `01-pseudocodigo` cuando puedas explicar tus soluciones con tus propias palabras.

## Criterio de avance

Este módulo se considera logrado cuando puedes:

- Explicar un problema simple antes de resolverlo.
- Separar entrada, proceso y salida.
- Proponer casos de prueba básicos.
- Crear carpetas con nombres claros.
- Registrar avances con Git.
- Escribir una evidencia breve de aprendizaje.
- Citar una fuente externa cuando uses material que no fue creado por ti.
- Declarar el uso de IA cuando corresponda.
- Reconocer cuándo necesitas descansar o pausar.
